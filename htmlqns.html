what is d/f b/w block elements vs inline elements?
Inline elements These elements do not start on a new line and only take up as much width as needed. They are typically used to define small pieces of content, like text or images, that are integrated into a larger block of text. Examples of inline elements include , input, label, select, and anchor. Block elements These elements always start on a new line and take up the full width available. They are typically used to define larger blocks of content, like paragraphs, headings, and lists. Examples of block elements include div, nav, header, section, footer, and ul.
what is semantic vs non-semantic elements?
semantic tags define the meaning of the content they contain, while non-semantic tags don't: Semantic tags These tags clearly describe the purpose of the content they contain, making it easier for both developers and browsers to understand the content. Examples of semantic tags include article, footer, header, and nav. Non-semantic tags These tags, such as div and , don't provide any information about the content they contain. Instead, they serve as containers for scripting or styling purposes.
d/f b/w div and span?
Div A block-level element that groups larger blocks of content, like paragraphs or sections, together. Divs are used for layout purposes and don't have semantic meaning. Span An inline element that groups smaller blocks of content, like words or short strings, together. Spans can be used for styling content, or to group elements that share attribute values.
